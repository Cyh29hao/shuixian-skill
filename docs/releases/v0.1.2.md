# 水仙.skill v0.1.2

> 这一版重点补的是导入后的“中间层”。  
> 不再只靠生吃 transcript，而是先把关系和价值观线索整理成一份可读画像。

## 这一版新增

- `tools/mirror_profiler.py`
  把 normalized transcript 整理成一份关系画像 / 价值观线索 / 节奏建议报告。
- `knowledge/analysis/`
  生成好的镜像目录现在会预留分析结果的位置。
- README / quickstart / skill workflow 更新
  明确推荐“导入 -> 画像报告 -> 再写镜像”的使用路径。

## 它解决什么问题

当用户导入的聊天记录一多，直接把 transcript 全塞给模型，效果很容易出现两个问题：

1. 关系边界混在一起，镜像会误把朋友、亲人、crush、前任的说话方式搅成一锅。
2. 价值观线索埋在长日志里，模型知道很多字，但不一定抓得住真正重要的雷区和共识。

`mirror_profiler.py` 的目标不是代替最终判断，而是先把这些原始材料压成一层更清楚的中间报告。

## 推荐新流程

1. 先导入 transcript 或微信聊天
2. 先跑 `contact-report` 挑重点关系
3. 再跑 `mirror_profiler.py` 生成画像报告
4. 最后用这份报告去构建或更新 mirror

## English Summary

`v0.1.2` adds a new middle layer:

- profile transcripts before building the mirror
- surface likely relationship types and topic clues
- keep raw logs from overwhelming the final mirror behavior
