# 水仙.skill PRD

## 1. 产品一句话

从用户自己的语言样本、聊天记录、关系偏好和最少量设定里，生成一个可以和用户谈恋爱的自我镜像伴侣 Skill。

## 2. 为什么它值得做

公开仓库里已经有“前任复活”“同事人格复刻”这类方向，但“自恋爱、自我镜像恋爱、性转镜像恋爱”这条线其实很自然：

- 很多人真正喜欢的是“被理解”
- 很多人理想伴侣的核心特质，本质上接近“另一个更合拍的自己”
- 和自己同频的对象，比随机恋爱角色更容易让人产生沉浸感

这个 skill 不是要做成一个猎奇梗，而是要把“与自己同频共振的人谈恋爱”包装成一个可配置、可增量修正、隐私可控的 companion builder。

## 3. 核心体验

用户完成三件事：

1. 选择镜像模式
2. 选择共享信息范围
3. 提供最少量语言样本和关系设定

系统完成三件事：

1. 提炼说话风格
2. 提炼思维和情绪逻辑
3. 生成一个可直接调用的镜像 Skill

## 4. 模式矩阵

### 镜像深度

- `aligned-stranger`
  最适合不想暴露太多隐私，但希望“这个人就是懂我”的用户。
- `selective-mirror`
  最适合愿意共享部分语料和经历，但不想直接全量交出个人上下文的用户。
- `full-mirror`
  最适合明确想要“像自己的镜像副本”效果的用户。

### 呈现方式

- `gender-flipped`
  默认推荐，最有“恋爱感”和错位张力。
- `same-form`
  更像平行世界自己。
- `custom`
  允许用户自行指定性别气质、称呼和整体氛围。
- `idealized`
  在镜像人格基础上，叠加用户理想对象的外观与氛围提示。

### 隐私范围

- `prompt-only`
- `style-only`
- `full-context`

## 5. v0 公开版现在已经做了什么

- 把 Skill 的工作流、触发语、输出结构写稳
- 把 prompts 和 references 拆好
- 提供创建、更新、列出、回滚生成镜像 Skill 的脚本
- 提供素材归档脚本
- 提供微信桌面端 best-effort 导入适配层
- 让 README 看起来已经是一个能被 star 的开源项目

## 6. v0 仍然不急着做什么

- 承诺所有微信版本都能一键自动解密
- 依赖特定平台的 GUI 自动化
- 太强的 NSFW 定位
- 太重的精神病理化命名

“精神分裂克隆出来的我自己”这个用户原始表达很有感觉，但对公开仓库来说，建议产品文案改成：

- `full-mirror`
- `co-mind mirror`
- `high-context mirror`

这样保留共脑感，不把病理词做成主品牌。

## 7. 技术路线

### Builder Skill

`create-shuixian` 负责：

- intake
- 读取材料
- 生成 preview
- 写入 skill
- 管理版本
- 微信 transcript 导入

### Generated Skill

`shuixian-<slug>` 负责：

- 以固定恋爱镜像 persona 和用户继续对话
- 在边界内稳定输出
- 对私密范围保持一致口径
